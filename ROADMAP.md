# DistributionOS Roadmap

## Purpose

This roadmap defines the staged path from project operating system to AI-powered distribution platform.

The roadmap is optimized for:

- solo founder execution
- fast validation with CareerScore
- minimal infrastructure burden
- durable architecture
- AI-assisted development

## Roadmap Strategy

DistributionOS should evolve in this order:

1. Project operating system
2. CareerScore manual growth OS
3. Internal product MVP
4. Agent-assisted workflows
5. Automated distribution workflows
6. Multi-customer platform

Manual clarity comes before automation.

## Phase 0: Project Operating System

### Goal

Create the durable planning, memory, and architecture foundation for the project.

### Deliverables

- `PROJECT_MEMORY.md`
- `ARCHITECTURE.md`
- `ROADMAP.md`
- `TASKS.md`
- recommended folder structure
- architecture principles
- database architecture
- deployment architecture
- agent architecture

### Success Criteria

- The project has a clear operating model.
- An AI agent can understand the project direction from markdown alone.
- No application code is required to understand the next steps.

### Status

In progress.

## Phase 1: CareerScore Manual Growth OS

### Goal

Use DistributionOS manually to plan and run CareerScore growth work.

### Deliverables

- CareerScore product profile
- CareerScore positioning
- ICP definition
- persona map
- channel strategy
- campaign planning template
- experiment tracker
- weekly review process
- first 10 growth experiments

### Core Questions

- Who exactly is CareerScore for?
- What pain does it solve?
- Which channels are most promising?
- What growth loops can be tested quickly?
- What assets are needed?
- What metrics matter weekly?

### Success Criteria

- CareerScore has a clear weekly growth process.
- Campaigns and experiments are planned in a repeatable format.
- Learnings are captured after each experiment.
- The founder can decide the next growth action with less ambiguity.
